# ATLAS Decision Log - Strategic Decisions & Rationale

**Purpose:** Document strategic decisions as you make them
**Status:** EMPTY - Fill when major decisions are made
**Updated:** Continuously (as decisions happen)

---

## Instructions

This file starts EMPTY. You fill it WHEN you make strategic decisions.

**When to add entries:**
- Major technical architecture choices
- Scope changes or feature additions/removals
- Strategic pivots or direction changes
- Trade-off decisions with significant impact

**Use ATLAS Mode 6 (Document Decision) to create well-formatted entries.**

---

## Entry Format

```markdown
## [Date]: [Effort/Context] - [Decision Title]

**Context:** [What situation led to this decision?]

**Decision:** [What did we decide?]

**Alternatives Considered:**
1. **[Option 1]**
   - Pros: [Benefits]
   - Cons: [Costs]
2. **[Option 2]** (CHOSEN)
   - Pros: [Benefits]
   - Cons: [Costs]
3. **[Option 3]**
   - Pros: [Benefits]
   - Cons: [Costs]

**Rationale:** [Why we chose this option]

**Trade-Offs Accepted:**
- [Trade-off 1]
- [Trade-off 2]

**Evidence/Input:** [Data, research, or input that informed decision]

**Validation Plan:** [How will we know if this was right?]

**Reversibility:** [HIGH/MEDIUM/LOW - how hard to undo?]

**Affected Steps:** [Which Steps does this impact?]

---
```

## Integration with Learnings

**Decisions ← Learnings:**
- Often triggered by learnings ("Based on [Learning], we decided [Decision]")
- Reference specific LEARNINGS_LOG entries when relevant

**Decisions → Future Planning:**
- Future AI1/AI2 read DECISION_LOG to understand past choices
- Prevents re-litigating settled decisions
- Provides context for why things are the way they are
